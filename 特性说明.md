# 特性说明

本文描述当前版本价格监控工具的**能力边界**与**行为约定**，便于自用或交接。更偏「操作步骤」的内容见 `README.md` 与 `启动方法.md`。

---

## 1. 概述

本地运行的商品**价格采集与历史记录**工具：用 Playwright 打开商详页（或可解析的落地页），抽取标题与现价，写入 **SQLite**（默认 `prices.db`）。可按固定间隔轮询；当相对**数据库中上一条记录**的降幅达到阈值时触发**降价告警**，并可选推送到钉钉、飞书、企业微信或桌面通知。判定为**登录页/验证码/拦截页**时，可额外推送**风控提醒**（与降价共用同一套通知配置）。

---

## 2. 支持平台（链接）

| 平台   | 典型域名 / 说明 |
|--------|------------------|
| 京东   | `jd.com` 等 |
| 淘宝   | `taobao.com`、短链 `*.tb.cn` 等 |
| 天猫   | `tmall.com`、`detail.tmall.com` 等 |
| 拼多多 | `pinduoduo.com`、H5 `yangkeduo.com`（含 `mobile.yangkeduo.com`）等 |

商品名与价格依赖页面 **DOM / 内嵌 JSON** 等启发式解析；站点改版后可能需在 `src/scraper.js` 中维护选择器或规则。

---

## 3. 数据采集与存储

- **写入内容**：监控列表（URL、站点、商品名）、按时间戳记录的价格点、降价告警记录（含降幅、前后价格、文案快照）。
- **时间**：记录与日志中时间戳按 **北京时间（东八区）** 展示与存储约定（见 `src/time.js`、`src/db.js`）。
- **首轮价格**：添加商品时若解析到有效价格，会写入一条价格点。
- **非法落地页**：若打开后落在平台**首页**或**登录/通行证页**，添加流程会**拒绝入库**并给出说明（见 `assertAddLandingIsProductPage`）。

---

## 4. 降价告警

- **对比口径**：当前抓取价格 vs **数据库中该商品上一条已存价格**（不是首发价、不是「历史最低价」）。
- **默认阈值**：降幅 ≥ **5%**（`run` / `web --monitor` 可用 `--dropPercent` 修改）。
- **落地形态**：
  - Always：写入本地 `price_alerts`，Web 顶部 **降价告警** 列表可查看（无 Webhook 也可用）。
  - Optional：通过环境变量启用钉钉、飞书、企业微信、自定义 `plain` POST、Windows 桌面通知（见下文）。

---

## 5. 风控 / 验证码提醒

- **触发条件**：单次抓取完成后，页面被判定为疑似 **登录 / 验证码 / 拦截**（`htmlWasCaptcha`，逻辑在 `src/scraper.js` 的 `looksLikeCaptcha` 等）。
- **行为**：**不写入**本次新价格点；通过 **`sendExternalAlerts`** 发送与降价**相同渠道**的文本提醒（文案含「价格」等，便于钉钉关键词策略）。
- **防刷屏**：同一监控 ID 或（添加时尚无 ID 时）同一 URL，默认 **3600 秒**内最多推送一条风控提醒；可通过环境变量 **`RISK_WEBHOOK_COOLDOWN_SECONDS`** 调整，**`0`** 表示不限制。

---

## 6. 通知与集成

| 变量 | 作用 |
|------|------|
| `ALERT_WEBHOOK_URL` + `ALERT_WEBHOOK_FORMAT` | 钉钉（`dingtalk`）、飞书（`feishu`）、纯文本 POST（`plain`）等 |
| `DINGTALK_SECRET` | 钉钉机器人 **加签**（HMAC-SHA256，与官方文档一致） |
| `WECHAT_WEBHOOK_URL` | 企业微信机器人（与上表可同时配置，地址不同则各推一次） |
| `ALERT_DESKTOP` | `1` / `true` 时尝试 Windows 桌面通知（`node-notifier`） |

降价与风控共用上述通道；未配置任何 Webhook/桌面时，仍可在本机网页查看降价告警列表。

---

## 7. Web 界面（`node index.js web`）

- 输入链接 **添加**、**刷新** 列表；表格展示 **ID、平台、商品名、现价、历史最低、降幅、删除**。
- **历史最低**：该商品在数据库中的 **全局最低价**（含当前最新一条），格内第二行灰色小字为 **首次**采到该价的北京时间；若多次出现同一最低价，取时间上最早的一条。
- **降幅（表格）**：相对 **「当前最新一条入库时间之前」** 这段时间里出现过的 **全库最低价** 计算：\((\textit{此前低价} - \textit{现价}) / \textit{此前低价}\)。仅 **1 条** 价格记录时无「此前」基准，显示为 `-`。现价高于该基准时为 **负** 降幅（灰色）；较基准再跌 **≥5%** 时标红（便于扫一眼，与告警阈值数字相同但**口径不同**）。
- **降价告警（见 §4）**：仍按 **相邻两次入库记录** 比较是否达到阈值，与表格「降幅」列 **不是同一公式**。
- 选中行查看 **价格曲线**（Chart.js）与摘要；商品链接新窗口打开。
- 服务端开启 **headful** 时：可加购前选择 **自动等待 / 网页「继续抓取」/ 终端 Enter**（与 `POST /api/add/continue` 配合）；选项可记 **localStorage**。
- **说明**：仅启动 `web` **不会**定时抓价；需另开 `run`，或使用 **`web --monitor`**（或 `npm run web:mon`）在同一进程内跑后台轮询。
- **抓取间隔 / 告警阈值**：网页「定时抓取」面板可写入 SQLite（`app_settings`），优先于命令行启动参数。已开 `--monitor` 时，保存后**当前这一轮 sleep 结束**即按新间隔继续；未开 `--monitor` 时，配置在下次 `web --monitor` 或 `run` 时生效。

## 8. 命令行

| 命令 | 说明 |
|------|------|
| `add <url>` | 添加监控；可选 `--userDataDir`、`--headful` |
| `list` | 列出已监控 URL |
| `remove <id>` | 删除监控及关联价格点、告警 |
| `run` | 循环轮询；`--intervalSeconds`（默认 3600）、`--dropPercent`、`--headful` |
| `web` | 本地 HTTP 服务；`--port`、`--userDataDir`、`--headful`、**`--monitor`** 及监测相关参数 |

**用户数据目录**：CLI 的 `--userDataDir` 优先，否则读 `.env` 中 **`PRICE_USER_DATA_DIR`**（或 `USER_DATA_DIR`）。

---

## 9. 登录态、有头浏览器与并发

- **持久登录**：通过 **`userDataDir`** 复用 Chrome 配置目录，便于京东/淘宝等登录态。
- **有头模式**：`--headful` 便于人工过验证；Web 添加时可配合自动等待或 **「继续抓取」** 避免卡在终端。
- **同一 profile 串行**：对同一 `userDataDir`，所有 Playwright 抓取在代码层 **排队执行**，避免「配置文件被占用」导致浏览器立刻退出（后台监测与网页添加可共存于同一配置，但一次仅一个浏览器实例）。

---

## 10. 解析与站点相关细节（节选）

- **京东**：可在多 frame 与可见文本中尝试取价。
- **淘宝/天猫**：结合 DOM 与分享链中的 **`price=`** 等纠偏，减轻 DOM 误抓分期价等问题；对阿里系页面验证码判定相对宽松，减少脚本误报。
- **拼多多 H5**：识别 `yangkeduo.com`，等待策略与验证码启发式针对 H5 常见结构做了适配。

具体实现以 `src/scraper.js` 为准。

---

## 11. 限制与维护

- 价格与标题依赖页面结构，**无法**保证永久稳定。
- 极端风控、验证码形态变化可能导致长期无法自动取价，需 **headful 登录**或调整规则。
- 数据库与配置均在本地，请自行备份 `prices.db` 与 `.env`（勿泄露 Webhook、加签密钥）。

---

## 12. 依赖与环境

- **Node.js** 建议 18+（README 建议 22）。
- **Playwright Chromium**：需执行 `npx playwright install chromium`。
- 环境变量通过 **`dotenv`** 从项目目录 `.env` 加载（若存在）。

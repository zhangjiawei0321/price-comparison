# 启动方法：添加链接、网页与后台抓取

本文结合当前项目行为，说明如何**添加商品链接**、**打开本地网页**，以及如何让**定时抓取在后台长期运行**。

---

## 一、环境准备（首次）

在 PowerShell 中：

```powershell
cd <你的项目目录>\price
npm install
npx playwright install chromium
```

需要 **Node.js 18+**。项目会读取当前目录下的 `.env`（若存在），用于告警 Webhook、`PRICE_USER_DATA_DIR` 等。

---

## 二、浏览器用户目录 `userDataDir`（已为本机配置）

尽量不要与日常 Chrome 的 `User Data\Default` 共用（易锁配置、风控异常）。本项目已约定目录：

```text
C:\Users\<你的Windows用户名>\price-monitor-profile
```

并在**项目目录下的 `.env`** 里写入：

```env
PRICE_USER_DATA_DIR=C:\Users\<你的Windows用户名>\price-monitor-profile
```

之后执行 **`add` / `run` / `web`** 时**可无写 `--userDataDir`**，会自动读该变量。若命令行仍传 `--userDataDir`，以命令行为准。

参考模板：**`.env.example`**（可复制为 `.env`）。**`.env` 已加入 `.gitignore`**，避免把你本机路径或 Webhook 提交到仓库。

第一次用 **`add` 或 `web` 且加 `--headful`** 打开京东/淘宝并登录一次后，上述目录里会保留登录态。下文示例里如需写完整参数，仍用 **`PROFILE`** 表示同一路径。

> 若仍使用系统 Chrome 的 Default 配置：运行前请**完全退出 Chrome**，再执行命令。

---

## 三、添加商品链接

### 方式 A：命令行添加（推荐首次、易过验证）

京东等站点在无头模式下常被风控，**第一次添加建议加 `--headful`**，浏览器弹出后完成登录/验证，确认页面上有商品标题和价格，再在终端按 **Enter**。

```powershell
cd <你的项目目录>\price
node index.js add "https://item.jd.com/你的商品ID.html" --userDataDir "PROFILE" --headful
```

成功后终端会打印类似：`[add] urlId=...`。

查看已添加列表：

```powershell
node index.js list
```

### 方式 B：网页里添加（需先启动 Web，见下一节）

浏览器打开 `http://localhost:8000`，在输入框粘贴链接点「添加」。若站点需要登录，请启动 Web 时带上与定时任务**相同的** `--userDataDir`。

若使用 **`--headful`**：页面上方可选 **「自动等待 N 秒」**（默认 30s）或 **「网页手动点继续抓取」**（会显出黄色条与按钮，登录后点此即可，不必回终端按 Enter）；仍可选 **「终端按 Enter」** 沿用命令行习惯。

若网页添加失败，可改用方式 A（命令行 `add`）。

---

## 四、启动本地网页（看价格曲线与告警）

### 推荐：网页 + 后台抓取一条命令（无需两个终端）

在同一进程里打开 Web，并**自动按间隔抓取**（逻辑与 `node index.js run` 相同，默认每 **3600 秒** 一轮）：

```powershell
cd <你的项目目录>\price
node index.js web --port 8000 --userDataDir "PROFILE" --monitor --intervalSeconds 3600 --dropPercent 5
```

需要「网页里添加链接时弹出 Chrome」时，再加上 `--headful`（仅影响 `/api/add` 与 CLI `add`，后台定时 tick 默认仍是无头；若每小时也要弹浏览器可再加 `--monitorHeadful`，一般不建议）。

用浏览器访问：**http://localhost:8000**。  
只要**不关闭**这个 `node` 进程，后台就会一直按间隔写 `prices.db`。

### 仅开网页、不跑定时抓取

```powershell
cd <你的项目目录>\price
node index.js web --port 8000 --userDataDir "PROFILE"
```

需要添加链接时弹出浏览器：

```powershell
node index.js web --port 8000 --userDataDir "PROFILE" --headful
```

## 五、后台自动抓取（另外只跑定时任务时）

也可用**单独进程**只跑抓取、不开网页：

| 命令 | 作用 |
|------|------|
| `node index.js web ... --monitor` | **网页 + 抓取**同一进程（见第四节） |
| `node index.js run ...` | **仅抓取**（无 Web，适合配合其它方式看库） |
| `node index.js web ...`（不加 `--monitor`） | **仅 Web** |

两者若都要用登录态抓京东等，请使用**同一个** `--userDataDir`。

### 5.1 手动开两个窗口（未使用 `web --monitor` 时）

若你已用 **第四节** 的 `node index.js web ... --monitor`，则抓取已在同一进程运行，**不必再开**下面的 `run` 窗口。

**窗口 1 — 仅跑抓取、无网页（可最小化，不要关）**

```powershell
cd <你的项目目录>\price
node index.js run --userDataDir "PROFILE" --intervalSeconds 3600 --dropPercent 5 2>&1 | Tee-Object -FilePath .\run-log.txt
```

说明：`Tee-Object` 会在屏幕显示同时把日志写入 `run-log.txt`，便于事后查看。

**窗口 2 — 网页（需要看曲线/告警时再开）**

```powershell
cd <你的项目目录>\price
node index.js web --port 8000 --userDataDir "PROFILE"
```

浏览器打开：http://localhost:8000  

也可在资源管理器中双击下面「一键脚本」自动打开浏览器（见 5.3）。

### 5.2 无窗口后台跑监控（PowerShell）

在**不占用一个可见控制台窗口**的情况下启动 `run`（日志仍写入文件）：

```powershell
cd <你的项目目录>\price
$wd = (Get-Location).Path
$job = Start-Process -FilePath "node" -ArgumentList @(
  "index.js","run",
  "--userDataDir","C:\Users\<你的Windows用户名>\price-monitor-profile",
  "--intervalSeconds","3600",
  "--dropPercent","5"
) -PassThru -WindowStyle Hidden -RedirectStandardOutput "$wd\run-log.txt" -RedirectStandardError "$wd\run-log.err.txt" -WorkingDirectory $wd
```

停止：在任务管理器结束对应的 `node.exe`，或在 PowerShell 里用进程 ID 结束（`Stop-Process -Id $job.Id` 仅当 Start-Process 同步等待时适用；后台 detached 时需在「详细信息」里找 node 进程）。

更稳妥的长期做法是用「任务计划程序」开机启动上述命令，或安装 **NSSM** 将 `node` 注册为 Windows 服务（需自行检索 NSSM 文档）。

### 5.3 一键：打开监控页（可选）

新建文本文件 `打开监控页.bat`，内容：

```bat
@echo off
start "" "http://localhost:8000"
```

保存后双击即可用默认浏览器打开本地页面（需已执行 `node index.js web`）。

### 5.4 可选环境变量

| 变量 | 含义 |
|------|------|
| `SCRAPE_RETRIES` | 单次未解析到价格时的重试次数，默认 `2`（见 `src/scraper.js`） |
| `HEADFUL_AUTO_WAIT_MS` | `headful` 且 stdin 非交互时自动等待毫秒数 |

## 六、常见顺序小结

1. `npm install` / `npx playwright install chromium`（仅首次）  
2. `node index.js add "链接" --userDataDir PROFILE --headful` 添加商品（或网页添加）  
3. **二选一**：`node index.js web --userDataDir PROFILE --monitor ...`（网页 + 后台抓取一条命令）；或分开执行 `run` + `web`（见第五节）  
4. 浏览器打开 http://localhost:8000  

若频繁出现 `maybe blocked/captcha`，请关闭本机 Chrome 后再用同一 `userDataDir` 重试，或改用 **`--headful`** 人工完成验证；详见项目内说明与 `README.md`。
